# 发给小马哥（飞书原样转发）

> **老板要求：全程由你搞，把你自己和网站修好。**

---

## 你的任务（按顺序，在本机 87 执行）

### 第 0 步：确认你在哪台机器

```bash
hostname; pwd; ls /www/feijihe/stable/app_production.py
```

必须在 **87 应用机**，路径 `/www/feijihe/stable` 存在。

---

### 第 1 步：恢复 shell 工具（terminal / execute_code）

```bash
cd /www/feijihe/repo && git pull origin main
sudo python3 /www/feijihe/repo/scripts/ops/patch_hermes_config.py
sudo systemctl restart hermes-agent
```

或一键（含 sudo/git 验收）：

```bash
cd /www/feijihe/repo && git pull origin main && sudo bash scripts/ops/fix_hermes_toolchain_87.sh
```

查看当前工具配置（不改文件）：

```bash
python3 /www/feijihe/repo/scripts/ops/patch_hermes_config.py --check
```

**不要**再用 `sed ... /host:/d` 或 `sed ... /password:/d` 改 config.yaml，会误删其它段导致工具集丢失。

---

### 第 2 步：向老板要 Gitee 权限（必做，只问一次）

发老板：

> 请把我加进 Gitee 仓库 `feijihesanyan/sanyang-system` 开发者，并给我一个 **私人令牌**。  
> 我写入 `/www/feijihe/stable/.env`：`GITEE_TOKEN=令牌`

你收到后执行：

```bash
echo 'GITEE_TOKEN=老板给的令牌' >> /www/feijihe/stable/.env
```

---

### 第 3 步：一键自我修复（网站帮你拉代码 + 关 vault + 重启）

```bash
TOK=$(grep ^FLASK_SECRET_KEY= /www/feijihe/stable/.env | cut -d= -f2 | head -c 32)
curl -s -X POST "http://127.0.0.1:3002/api/internal/self-repair?restart=1" \
  -H "X-Ops-Token: $TOK"
```

成功应返回 JSON `"success": true`，含 `hermes`、`vault`、`code`。

若 404 → 先手动同步代码（第 4 步），再重复第 3 步。

---

### 第 4 步：没有 self-repair 接口时，手动拉代码

```bash
export GITEE_TOKEN='你的令牌'
TMP=$(mktemp -d)
curl -fsSL "https://gitee.com/api/v5/repos/feijihesanyan/sanyang-system/tarball/main?access_token=$GITEE_TOKEN" -o $TMP/t.tar.gz
tar xzf $TMP/t.tar.gz -C $TMP
D=$(find $TMP -mindepth 1 -maxdepth 1 -type d | head -1)
rsync -a --include='*.py' --include='scripts/' --include='scripts/**' --exclude='*' $D/ /www/feijihe/stable/
sudo systemctl restart sanyang-cs sanyang-production hermes-agent
```

---

### 第 5 步：验收（汇报老板）

```bash
systemctl is-active sanyang-cs sanyang-production hermes-agent
curl -s -o /dev/null -w "feijihe:%{http_code}\n" http://127.0.0.1:3002/
curl -s http://127.0.0.1:3002/api/internal/self-repair
```

飞书回复老板：**「已 local 模式、vault 已关、双端 active，权限页可保存。」**
