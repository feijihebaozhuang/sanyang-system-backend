# 小马哥工具链为什么老被锁死？怎么一次性打开？

> **小马哥 = 87 上的 Hermes 智能体**，不是真人。  
> 他说「工具链被 SSH 卡死」= **配置错了 + 权限不够**，不是代码 bug。

---

## 一句话原因

他原来配成 **`backend: ssh`**（远程 SSH 去别的机器跑命令），但：

| 卡点 | 结果 |
|------|------|
| 213 SSH 已锁、8 月要退 | 跳板没了 |
| 87 没配密钥 / 密码错 | SSH 连不上 |
| `sudo` 要密码 | 智能体不能交互，命令卡死 |
| Gitee 私有库没令牌 | `git pull` 失败 |
| `/www/feijihe/repo` 不存在 | deploy 没地方跑 |

所以不是「再给他开一个 SSH 就行」，而是 **改成在本机跑命令 + 补权限**。

---

## 正确状态（打开后应是这样）

| 项 | 应为 |
|----|------|
| Hermes `config.yaml` | **`backend: local`**（在 87 本机执行） |
| 进程位置 | **87**，不在 213 |
| admin sudo | **NOPASSWD**（免密） |
| 拉代码 | **GITEE_TOKEN** 或仓库开发者权限 |
| 部署目录 | `/www/feijihe/repo` + `stable/` |

---

## 一次性解锁（你本人不用操作）

**找有大虾 / 能在阿里云 Workbench 登录 87 的人**，用 **root** 执行：

```bash
# 若已有 Gitee 私人令牌（小马哥或老板账号生成）：
export GITEE_TOKEN='私人令牌'
curl -fsSL "https://gitee.com/api/v5/repos/feijihesanyan/sanyang-system/tarball/main?access_token=${GITEE_TOKEN}" -o /tmp/s.tar.gz
tar -xzf /tmp/s.tar.gz -C /tmp
DIR=$(find /tmp -mindepth 1 -maxdepth 1 -type d | head -1)
sudo bash "$DIR/scripts/ops/unlock_hermes_87.sh"
```

或 repo 已在 87 上时：

```bash
sudo GITEE_TOKEN='私人令牌' bash /www/feijihe/repo/scripts/ops/unlock_hermes_87.sh
```

脚本会自动：

1. Hermes **SSH → local**
2. 删掉 config 里的 host/port/password
3. **admin 免密 sudo**
4. 写入 **GITEE_TOKEN**（若提供）
5. **重启 hermes-agent**

## 老板只需做 2 件（网页，30 秒）

1. Gitee → `feijihesanyan/sanyang-system` → **成员** → 加运维用的 Gitee 账号为开发者  
2. 把上面 **Workbench 解锁命令** 发给 **大虾**（或任何能进 87 Workbench 的人）跑一遍  

**你不用** scp、不用本地 Git、不用 SSH 回家连 87。

飞书 @小马哥 让他执行（或他自己跑）：

详见 `docs/小马哥-87部署必读.md`。

- ❌ 在 config.yaml 里写 SSH **密码**（会泄露且易过期）  
- ❌ 213 上对公网开 22 当跳板  
- ❌ 指望 C 把 py 文件 scp 到 `/tmp`（代码只在 Gitee）
