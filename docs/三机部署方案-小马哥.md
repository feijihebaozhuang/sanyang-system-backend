### 5.4 Nginx 配置

文件示例：`/etc/nginx/conf.d/feijihe.conf`

```nginx
# 生产端 feijihe.top → 3002
server {
    listen 443 ssl http2;
    server_name feijihe.top www.feijihe.top;

    ssl_certificate     /path/to/feijihe.top.pem;
    ssl_certificate_key /path/to/feijihe.top.key;

    location / {
        proxy_pass http://127.0.0.1:3002;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 120s;
    }
}

# 客服端 zean.feijihe.top → 3001
server {
    listen 443 ssl http2;
    server_name zean.feijihe.top;

    ssl_certificate     /path/to/feijihe.top.pem;
    ssl_certificate_key /path/to/feijihe.top.key;

    location / {
        proxy_pass http://127.0.0.1:3001;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 120s;
    }
}

# HTTP 跳 HTTPS
server {
    listen 80;
    server_name feijihe.top www.feijihe.top zean.feijihe.top;
    return 301 https://$host$request_uri;
}
```

### 5.8 NOPASSWD sudo（admin 免密，便于 systemctl / deploy）

> **现状**：admin 用户 `sudo` 要密码，`deploy.sh` 里 `systemctl restart` 会卡住。  
> **必须在应用机上用 root 或有 sudo 密码的账号执行一次**（本地 Cursor 无法代跑 SSH）。

#### 方案 A：全免密（小马哥推荐，最简单）

SSH 登录 **8.166.132.87**，用 **root** 或能输入 sudo 密码的账号，**整段复制执行**：

```bash
# 1. 写入 sudoers 片段（注意文件名不能含 . 以外的特殊字符）
echo 'admin ALL=(ALL) NOPASSWD: ALL' | sudo tee /etc/sudoers.d/admin

# 2. 权限必须是 0440，否则 sudo 会忽略该文件
sudo chmod 0440 /etc/sudoers.d/admin

# 3. 语法校验（必须显示 parsed OK，否则不要关 SSH）
sudo visudo -c -f /etc/sudoers.d/admin

# 4. 验收（不应再要密码）
su - admin -c 'sudo -n systemctl status sanyang-production --no-pager | head -3'
su - admin -c 'sudo -n true && echo NOPASSWD_OK'
```

期望最后一行输出 `NOPASSWD_OK`。

#### 方案 B：仅放行 systemctl（更安全，若 A 不想用全免密）

```bash
sudo tee /etc/sudoers.d/admin-sanyang <<'EOF'
admin ALL=(ALL) NOPASSWD: /bin/systemctl, /usr/bin/systemctl
EOF
sudo chmod 0440 /etc/sudoers.d/admin-sanyang
sudo visudo -c -f /etc/sudoers.d/admin-sanyang
su - admin -c 'sudo -n systemctl status sanyang-cs --no-pager | head -2'
```

#### 常见失败原因

| 现象 | 原因 | 处理 |
|------|------|------|
| `visudo` 报 syntax error | 文件内容写错 | `sudo rm /etc/sudoers.d/admin`，重新执行方案 A |
| 仍要密码 | 文件权限不是 0440 | `sudo chmod 0440 /etc/sudoers.d/admin` |
| 仍要密码 | admin 不在 sudo 组 | `sudo usermod -aG wheel admin` 或 `sudo usermod -aG sudo admin`（视系统而定），重新登录 |
| 仍要密码 | 文件名含 `.` | 用 `admin` 或 `admin-sanyang`，不要用 `admin.conf` |
| `sudo: unknown user: admin` | 用户名不对 | `id admin` 确认用户名 |

#### 回滚

```bash
sudo rm -f /etc/sudoers.d/admin /etc/sudoers.d/admin-sanyang
```

### 6.2 从应用机导出 permission_data

```bash
# 在应用机 8.166.132.87 执行
cd /www/feijihe/stable
python3 -c "
import json
from pathlib import Path
d = json.loads(Path('data.json').read_text(encoding='utf-8'))
pd = d.get('permission_data', {})
Path('/tmp/permission_data.json').write_text(
    json.dumps(pd, ensure_ascii=False, indent=2), encoding='utf-8')
print('exported', len(pd), 'keys')
"
scp /tmp/permission_data.json admin@8.163.107.156:/opt/sanyang-config/permission_data.json
```

### 6.3 部署 vault 服务

```bash
# 156 上
scp 应用机:/www/feijihe/repo/scripts/permission_vault_server.example.py /opt/sanyang-config/
```

创建 systemd 单元 `/etc/systemd/system/sanyang-permission-vault.service`：

```ini
[Unit]
Description=Sanyang permission_data vault (read-only API)
After=network.target

[Service]
Type=simple
User=admin
WorkingDirectory=/opt/sanyang-config
Environment=PERMISSION_VAULT_BIND=0.0.0.0
Environment=PERMISSION_VAULT_PORT=9443
Environment=PERMISSION_VAULT_DATA_FILE=/opt/sanyang-config/permission_data.json
Environment=PERMISSION_VAULT_TOKEN=替换为长随机串
ExecStart=/usr/bin/python3 /opt/sanyang-config/permission_vault_server.example.py
Restart=on-failure
RestartSec=5

[Install]
WantedBy=multi-user.target
```

```bash
sudo systemctl daemon-reload
sudo systemctl enable sanyang-permission-vault
sudo systemctl start sanyang-permission-vault
```

```bash

mysqldump -u sanyang_app -p \
  --single-transaction --routines --triggers \
  sanyang | gzip > sanyang_to_rds_$(date +%Y%m%d).sql.gz

# === 2. RDS 控制台建库建用户（或 SQL）===
# CREATE DATABASE sanyang ...;
# CREATE USER 'sanyang_app'@'%' ...;  # 白名单由 RDS 安全组控制
# GRANT SELECT,INSERT,UPDATE,DELETE ON sanyang.* TO 'sanyang_app'@'%';

# === 3. 导入 RDS ===
gunzip -c sanyang_to_rds_YYYYMMDD.sql.gz | mysql -h rm-xxx.mysql.rds.aliyuncs.com -u sanyang_app -p sanyang

# === 4. 应用机改 .env（唯一必改项）===
# MYSQL_HOST=rm-xxx.mysql.rds.aliyuncs.com   # RDS 内网地址
# MYSQL_PORT=3306
# MYSQL_USER=sanyang_app
# MYSQL_PASSWORD=新密码
# MYSQL_DATABASE=sanyang

# === 5. 重启双端 ===
sudo systemctl restart sanyang-cs sanyang-production

# === 6. 验收 ===
# SELECT COUNT(*) FROM dimoldb;
# SELECT COUNT(*) FROM order_cache_orders;
# 浏览器打单 / 报价 / 刀模
```

## 十一、常见问题

### Q1：deploy 后报价/映射变了？

`deploy.sh` 不应覆盖 JSON。若变了，查是否有人手工 cp 了 repo 里的 example JSON，或 Python 启动时误写了配置。铁律：配置以服务器 JSON / MySQL 为准。

### Q2：MySQL 连不上「autocommit 重复」？

`settings.py` 里 `get_db_config()` 已有 `autocommit: True`；各模块 `connect` 时不要再传第二个 `autocommit=`。写库模块（如 `order_cache_store`）应 `pop("autocommit")` 后显式 `autocommit=False`。

### Q3：156 vault 挂了怎么办？

应用机会回退读本地 `data.json`（日志：`保险库拉取失败，回退 data.json`）。156 需监控 + 定期备份 JSON。

### Q4：systemctl 要 root 密码？

临时用 `sshpass` + sudo；长期配 5.8 节 NOPASSWD。

### Q5：213 内网 IP 和应用机连不上？

检查：同一 VPC、安全组 3306 入站仅应用机、MySQL `bind-address`、用户 `'sanyang_app'@'应用机IP'` 是否匹配。
