# 生产端恢复部署（给小马哥）

> 目标：刀模库恢复正常（18111 条）、页面不乱码、打单/库存/快麦同步能用。  
> **不用**整库回滚 Git，按下面做即可。

## 一、服务器上执行（复制整段给 root/admin）

```bash
set -e
REPO=/www/feijihe/repo
STABLE=/www/feijihe/stable

# 1. 拉最新代码
cd "$REPO"
git pull origin main

# 2. 备份刀模表（必做）
mysqldump -u"$MYSQL_USER" -p"$MYSQL_PASSWORD" sanyang dimoldb \
  > /home/admin/backup/feijihe/dimoldb_before_$(date +%Y%m%d_%H%M).sql

# 3. 同步 Python 并安装依赖
"$REPO/deploy.sh"

# 4. 覆盖前端（deploy 不会拷 html）
cp "$REPO/index.html" "$STABLE/index.html"
chown admin:sanyang "$STABLE/index.html"

# 5. 导入合并好的刀模（18111 条，约 1 分钟）
cd "$STABLE"
source venv/bin/activate
python3 "$REPO/scripts/import_dimoldb_merged_json.py" "$REPO/data/import/dimoldb/dimoldb_merged.json"

# 6. 重启生产端 3002（按你们现网方式，示例）
pkill -f "app_production.py" || true
sleep 1
cd "$STABLE"
nohup python3 app_production.py >> /tmp/app_production.log 2>&1 &
echo "3002 started"

# 7. 可选：重启客服 3001
# pkill -f "app_cs.py" || true
# cd "$STABLE" && nohup python3 app_cs.py >> /tmp/app_cs.log 2>&1 &
```

MySQL 账号密码用服务器 `stable/.env` 里的配置；`mysqldump` 也可在面板里手动导出 `dimoldb` 表。

## 二、部署后验收（浏览器）

1. 打开 https://feijihe.top ，**Ctrl+F5** 强刷  
2. **刀模库**：能分页，列表有名称+尺寸+编码，总条数约 **18111**  
3. **成品库存**：打开不长时间转圈  
4. **打单**：有订单列表；点一次「同步/刷新」拉快麦  
5. 随便 **编辑保存一条刀模** 应提示成功
