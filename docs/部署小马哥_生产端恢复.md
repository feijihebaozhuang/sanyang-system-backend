# 生产端恢复部署（给小马哥）

> 目标：刀模库恢复正常（18111 条）、页面不乱码、打单/库存/快麦同步能用。  
> **不用**整库回滚 Git，按下面做即可。

## 一、服务器上执行（复制整段给 root/admin）

```bash
set -e
REPO=/www/feijihe/repo
STABLE=/www/feijihe/stable

# 1. 拉最新代码
cd "$REPO"
git pull origin main

# 2. 备份刀模表（必做）
mysqldump -u"$MYSQL_USER" -p"$MYSQL_PASSWORD" sanyang dimoldb \
  > /home/admin/backup/feijihe/dimoldb_before_$(date +%Y%m%d_%H%M).sql

# 3. 同步 Python 并安装依赖
"$REPO/deploy.sh"

# 4. 覆盖前端（deploy 不会拷 html）
cp "$REPO/index.html" "$STABLE/index.html"
chown admin:sanyang "$STABLE/index.html"

# 5. 导入合并好的刀模（18111 条，约 1 分钟）
cd "$STABLE"
source venv/bin/activate
python3 "$REPO/scripts/import_dimoldb_merged_json.py" "$REPO/data/import/dimoldb/dimoldb_merged.json"

# 6. 重启生产端 3002（按你们现网方式，示例）
pkill -f "app_production.py" || true
sleep 1
cd "$STABLE"
nohup python3 app_production.py >> /tmp/app_production.log 2>&1 &
echo "3002 started"

# 7. 可选：重启客服 3001
# pkill -f "app_cs.py" || true
# cd "$STABLE" && nohup python3 app_cs.py >> /tmp/app_cs.log 2>&1 &
```

MySQL 账号密码用服务器 `stable/.env` 里的配置；`mysqldump` 也可在面板里手动导出 `dimoldb` 表。

## 二、部署后验收（浏览器）

1. 打开 https://feijihe.top ，**Ctrl+F5** 强刷  
2. **刀模库**：能分页，列表有名称+尺寸+编码，总条数约 **18111**  
3. **成品库存**：打开不长时间转圈  
4. **打单**：有订单列表；点一次「同步/刷新」拉快麦  
5. 随便 **编辑保存一条刀模** 应提示成功

- 不要改表头加「生产规格」「快麦商品映射」

## 四、出问题联系开发

- 刀模条数不对：`SELECT COUNT(*) FROM dimoldb;` 应为 18111  
- 页面乱码：确认 `stable/index.html` 已从 repo 复制且已强刷  

## 五、运维注意（2026-05 线上踩坑）

1. **3002 端口**：必须用 **admin** 用户的 systemd 服务跑生产端。若 `systemctl start` 失败，先查是否被 **root 旧进程** 占端口：  
   `ss -tlnp | grep 3002` → 勿直接 `pkill` 误杀，确认 PID 后再停；服务应以 admin 运行。  
2. **生产前端**：`git pull` 后务必 `cp repo/index.html → stable/index.html`；`rebuild_index_html.py` 会生成 index，不要用手改 cs 旧版覆盖首页刀模查询。  
3. **客服端 `app_cs.py`**：`/api/dimoldb/search` 需带 `infer_type_class` + 每条 `dim_type` 补全（与 `app_production.py` 一致）。部署时勿用旧包覆盖该函数。  
4. **验收**：生产首页搜 `40×36×8`（外径）应见「当前查询：外径」+ 组合刀模紫色「组合」标签；客服 `zean.feijihe.top` / `main_app.html` 搜索 API 返回项含 `dim_type`。
