## 一、一次性：你要有的权限（找老板 1 分钟）

| 权限 | 谁给 | 怎么做 |
|------|------|--------|
| **Gitee 仓库开发者** | 老板 | Gitee 网页 → `feijihesanyan/sanyang-system` → 管理 → 成员 → 加你的 Gitee 账号 |

**不要**向老板要 Gitee 登录密码。你自己在 Gitee → 设置 → **私人令牌** 生成一个（勾选 `projects` 读权限即可）。

## 二、首次部署（repo 坏了 / 不存在）

Workbench 登录 **87**，**整段复制执行**（把 `你的私人令牌` 换成你在 Gitee 生成的）：

```bash
export GITEE_TOKEN='你的私人令牌'
TMP=$(mktemp -d)
curl -fsSL "https://gitee.com/api/v5/repos/feijihesanyan/sanyang-system/tarball/main?access_token=${GITEE_TOKEN}" -o "$TMP/repo.tar.gz"
tar -xzf "$TMP/repo.tar.gz" -C "$TMP"
SOURCE_DIR=$(find "$TMP" -mindepth 1 -maxdepth 1 -type d | head -1)
export SOURCE_DIR
bash "$SOURCE_DIR/scripts/ops/bootstrap_repo_87.sh"
```

脚本会自动：

1. 从 Gitee 拉最新 `main` 到 `/www/feijihe/repo`
2. `./deploy.sh` 同步 `.py` 到 `/www/feijihe/stable`（**不覆盖** `.env`、`data.json`）
3. 若已配 `PERMISSION_VAULT_URL`，自动补 `PERMISSION_VAULT_WRITE_URL`
4. 重启 `sanyang-cs`、`sanyang-production`

## 三、以后每次 C 说「已 push」

```bash
export GITEE_TOKEN='你的令牌'
cd /www/feijihe/repo && git pull origin main && ./deploy.sh
sudo systemctl restart sanyang-cs sanyang-production
```

或再跑一遍 `bootstrap_repo_87.sh`（已存在 repo 时会走 pull）。

## 五、铁律（deploy 不会破坏配置）

- **只同步 `.py`**，不动 stable 里 admin 改好的 `data.json`、`quote_data.json`、`.env`
- **权限真源**在 **156 vault**（大虾管服务）；87 网页保存会 POST 到 156
- **213** 只管 MySQL，不要在那边拉业务代码

## 六、常见问题

| 问题 | 处理 |
|------|------|
| `GITEE_TOKEN` 无效 | 重新生成令牌；确认老板已加你为仓库成员 |
| 没有 `/www/feijihe/stable/.env` | 从旧机备份 scp，或复制 `.env.example` 后填 MySQL/密钥 |
| `git pull` 失败 | 用 `bootstrap_repo_87.sh` 重建 repo |
| 权限保存仍失败 | 检查 `.env` 三行 `PERMISSION_VAULT_*`；156 vault 是否 active |
